1次課題: ファッションECにおけるレコメンドエンジン

設問

ファッションECの特徴をうまく利用したエンジンを作る想定で以下の問いに答えてください。

  • ファッションアイテムを扱うECで観測される「購買ログ」にはどんな特性が存在すると思いますか？ 商材ごとにEC上におけるユーザーの購買行動は変化すると考えることは自然な発想です。観測されるデータの背後で起こっていると思われる現象と絡めて説明してください。 本問はレコメンドの経験を問うものではありません。自由に想像して回答してください。 (参考までに、一般的なECで得られるデータのサンプルを提示します。これのほか、実際のECで扱っていると思われるデータやSNSなど外部のデータソースと組み合わせて回答しても構いません。)

  • どんなアルゴリズムが有効だと思いますか？ 1.で説明した特性を利用したレコメンドアルゴリズムを開発し、ECの売上向上を目指します。 どんなアルゴリズムが有効だと思いますか？アルゴリズムの概要および効果的だと考える理由を説明して下さい。

  • 2.で説明したレコメンドアルゴリズムを実装してください。 一般に公開されているベンチマーク、もしくは自作したダミーデータを用いてアルゴリズムの稼働確認をして下さい。ただし、稼働確認に用いるデータはファッションに関係していなくても構いませんし、1.で説明した特性を満たしている必要もありません。 (本問は目的に対して手段を正しく選択できていることを重視するため、高度なアルゴリズムである必要はありません。目的に対して手段を正しく選択できるかを重視するため高度なアルゴリズムである必要はない。)

解答

ファッションアイテムを扱うEC上での購買ログにはどのような特徴が存在するのか？

  1. ユーザーは当初の目的であった優先順位の高い商品を買ってから、その後に付随して違うアイテムを探す。
  2. 性別、年代、地域によってユーザーの購買アイテムが大きく異なっていく。

自分はこの問いに対して、普段自分がどのような行動をEC上で行なっているか、ということを想起するところから始めた。自分は通常一つ以上の目的の商品を思い浮かべてからECを訪れている。ユーザーの中には、ウィンドーショッピング目的で訪れ、そこで気になった商品があるとクリックして購買するというユーザーも多いであろうが、自分にはそのケースは当てはまらない。そのため、今回の分析レポートの分析対象からはそのようなユーザーを外すことにした。

自分の場合、必ず一つの求めていた商品を見つけた後に、関連商品やその他セール対象になっている商品なども探し、複数個の商品をカートにプールして決済を行う。自分のこのようなEC上での行動経験をもとにファッションアイテムを扱うECで観測される「購買ログ」には以下の特徴が大きくあると考える。

ファッションアイテムを扱うEC上での購買ログにはどのような特徴が存在するのか？ 自分はこの問いに対して、普段自分がどのような行動をオンラインショッピングサイトで行なっているか、ということを想起するところから始めた。 自分は普段、ECサイトを訪れる際に、何か目的を持っている。中には、単なるウィンドーショッピング目的で訪れ、そこで気になった商品があるとクリックして購買するというユーザーも多いであろうが、自分にはそのケースは当てはまらないため、今回の分析レポートの分析対象からは外すとした。 自分の場合、必ず一つの求めていた商品を見つけた後に、関連商品やその他セール対象になっている商品なども探し、複数個の商品をカートにプールして決済を行う。 自分のこのようなECサイト上での行動経験をもとに以下の特徴が大きくあると仮定した。 ユーザーは最も優先順位の高い商品を買うのみではなく、それに関連した商品も探し、同時に決済することが多い。 以上の仮説をもとに分析レポートを作成していく。

さらにもう一つ特徴を挙げるとするならば、ユーザーの属性においても、大きく購買行動は異なると考えた。 この二つ目の仮説を確認するためには私はTwitterでファッションというワード含むツイートを抽出して、大まかなユーザーの属性を捉えることをした。

どんなアルゴリズムが有効か？

1.で説明した特性を利用したレコメンドアルゴリズムを開発し、ECの売上向上を目指します。 どんなアルゴリズムが有効だと思いますか？アルゴリズムの概要および効果的だと考える理由を説明して下さい。

この分析を行う上で効果的だと思われた手法はMarket Basket Analysisという手法である。 この手法をメリットとデメリットを明記し、他に選べたであろう分析手法となぜそれら選ばなかったのかについての理由を提示する。 そもそもMarket Basket Analysisとは何か？ Market Basket Analysisは商品同士の関連性を明らかにする為に小売大手によって用いられている重要な技術の一つである。主に、商品決済において頻繁に同時に取引される商品の組み合わせを明らかにすることによって当分析手法は効果を発揮する。言うなれば、当手法は顧客が買う商品の関連性を売り手側に (筆者日本語訳) To put it another way, it allows retailers to identify relationships between the items that people buy. Association Rules are widely used to analyze retail basket or transaction data, and are intended to identify strong rules discovered in transaction data using measures of interestingness, based on the concept of strong rules. 以上が、オンライン文献より引用した説明であるが、このレポートではこの分析手法を用いて、どの商品が同時に購入されることが多いか、を発見する。その後に、自分のように最も優先順位の高い商品を買ったユーザーに対して頻繁に同時に購入されている商品を提示しやすくし、さらなる購買を促すというレコメンドエンジンを作成する。

Support & Confidence

probability that a customer will buy two items (A & B) together

support_df <- read.csv("support_df.csv") 
#上記に使っているデータを整形して、購買商品の組み合わせを作ったデータフレームをインポート
# データ整形の過程は外部リンク参照(https://github.com/koki25ando/insurance/blob/master/cleasingofbread.R)
confidence <- function(first_item){
   pro_of_first_item <- Probability_df %>%
    filter(Var1 == first_item)
   Pobability_of_item  <- pro_of_first_item$Pro
   print(paste0("Probability of the item: ", Pobability_of_item))

   sup_con_dat  <- support_df[support_df$Var1 %like% first_item, ] %>%
     mutate(Confidence = Support/Pobability_of_item) %>%
     head(10)
   print(sup_con_dat)

   sup_con_dat$Var1 <- sup_con_dat$Var1 %>%
     str_remove(first_item) %>%
     str_remove(", ")
   print(sup_con_dat)
   pro_of_recommend_item  <- Probability_df %>%
     filter(Var1 %in% sup_con_dat$Var1)
   merge(sup_con_dat, pro_of_recommend_item, by = "Var1") %>%
     mutate(Lift = Confidence/Pro)
}

Limitations

  • 使用したデータは食品でありさらには、保存が効かない商品についてだった。その為季節性の激しいさらには長持ちする衣類を対象にすると思うような効果を出せない可能性もある。
  • さらには一般的に衣類はブランド品から日用品といったものまで幅広い価格帯の商品が多く、さらには季節性も激しいと思われる。

Challenges to overcome

  • 商品の価格帯が大きく変動するので、ユーザーごとにクラスタリングしたのちに、クラスターごとの購買ログから上述したパターン認識を行い商品をレコメンドするといった手法も実行可能ではありそう。実際のデータがあれば是非とも挑戦したいと思った。
  • このレポートで用いたように扱う商品が多くなればなるほどなかなか組み合わせというものは見つけづらいかも知れない。その為、サンプルデータで提供されていたような小カテゴリやブランドといった要因で同時に購入される商品の組み合わせを発掘して、レコメンドするという方法も可能であると思われる。
